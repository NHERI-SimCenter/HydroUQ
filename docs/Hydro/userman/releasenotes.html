6. Release Notes¶

6.1. Version 1.0¶

Version 1.0.0¶

Release date: Mar 31st, 2021

  1. Supports run on DesignSafe only. Local run on user’s desktop is not supported.

  2. Hydro-UQ v1.0.0 currently only supports SI (m, kg, s, K) units.

  3. Supports two-phase isothermal flows only. Water and air are considered as the two primary phases. However, this can be modified in the material properties to accommodate any other alternative two-phases instead.

  4. Simulation types:

    1. CFD to resolve SW (Using SW results), CFD using bathymetry data, CFD of wave flume are supported.

    2. CFD to resolve SW (Using SimCenter library), CFD with STL geometry, with maps and surrogate solver are currently not supported in v1.0.0

    3. For simulation type with SW-CFD coupling, v1.0.0 considers one point on the interface. However, more flexibility will be made available in the future versions.

  5. Geometry:

    1. Geometry can be imported as Bathymetry files (GeoClaw format - type 1 / 2 / 3). Wave flume geometry is generated using the in-build Hydro-Flume digital twin tool. Import of geometry in STL file format is not supported in v1.0.0

    2. Shallow-water to CFD interface can be imported as a .csv file only.

    3. Buildings of rectangular / square shapes are supported in v1.0.0. These can be provided in the table or can be generated parametrically. Importing buildings as a .csv file is not currently supported in v1.0.0 but will be added to upcoming versions. More complicated shapes will be added in the upcoming versions.

    4. Floating bodies are not supported in v1.0.0. Support will be added in upcoming versions.

  6. Meshing:

    1. Supports blockMesh and snappyHexMesh for internal meshing

    2. Suports import for the following mesh formats: Ansys Fluent (.msh), Ansys I-DEAS (.ans), CFX mesh (.geo), GAMBIT mesh (.neu), Gmsh mesh (.msh)

    3. Supports import of OpenFOAM / OlaFlow mesh dictionaries, namely the blockMeshDict and snappyHexMeshDict

  7. Materials:

    1. Supports Newtonian materials only.

    2. Supports kinematic viscosity and density of the two phases in addition to the surface-tension between the fluids.

  8. Initial conditions:

    1. For CFD simulations that resolve the shallow-water solutions, the initial conditions are derived from the shallow-water solutions.

    2. For all other simulation types, the user-specified initial conditions include velocity, pressure and phase.

  9. Boundary conditions:

    1. The boundary conditions can be selected based using standard or mesh-based. Here standard patches include (entry / exit / inlet / outlet / left / right. Mesh-based is based on patch names provided in the STL / mesh-dictionary files used to define the geometry / mesh.

    2. Standard velocity

    3. Wave maker input conditions are supported for input velocity

    4. Outlet velocity conditions supported are

    5. TO BE COMPLETED

  10. Domain decomposition:

    1. Allows simple and hierarchial decomposition techniques from OpenFOAM.

    2. Can set start and end times for simulation

    3. Can set time interval and the write intervals

    4. Restarting facility is supported

  11. Turbulence:

    1. Presently only RANS is supported for turbulence modeling. LES will be added to the upcoming versions.
